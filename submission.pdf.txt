## System Design Explanation
- **Signal Ingestion:** High-throughput, async API (FastAPI) accepts batched signals, queues for processing.
- **Debouncing:** Redis caches recent signals by source, groups within 10s window to a single incident.
- **Incident Management:** PostgreSQL stores incidents, signals, RCA. Status lifecycle enforced.
- **Async Processing:** RabbitMQ decouples ingestion from processing. Worker handles background jobs.
- **Frontend:** React + TailwindCSS dashboard, list, detail, RCA input, status updates.
- **Dockerized:** All services orchestrated via docker-compose for local development.

---

## Key Features
- **Debouncing:** Prevents duplicate incidents, groups signals by source/time window.
- **Scalability:** Async queue, stateless backend, horizontal scaling possible.
- **Fault Tolerance:** Retries, non-blocking ingestion, resilient to DB/queue slowness.
